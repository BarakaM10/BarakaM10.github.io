NextJS Website Migration Journey

The long anticipated website migration has started! Well, I've just cloned the repo and set up the environment, so we'll see how it goes. Moving from React to a NextJS framework is actually not that complicated. IT'S BACKEND TIME

Why NextJS?

Better indexing by search engines(Google)

  • Server-side rendering for better SEO and performance
  • Built-in routing system
  • Automatic code splitting and optimization
  • React-based component architecture

Project Setup

Getting started with NextJS was straightforward. Here's how I set up the development environment:

Step 1: Environment Setup

First, getting the latest version of Node.js installed and then created a new NextJS project:

Terminal 
                            
# Check Node.js version
node --version

# Create new NextJS project
npx create-next-app@latest school-website
cd school-website

# Install additional dependencies
npm install @next/font lucide-react

# Run development server
npm run dev

Step 2: Project Structure

NextJS comes with a well-organized project structure. Here's what I'm working with:

Project Structure 
                            
school-website/
├── src  
|   ├──app/
│   |    ├── globals.css
│   |    ├── layout.tsx
│   |    └── page.tsx
│   ├── components/
|   └── public/
│       ├── assets/
│       └── favicon.ico
├── package.json
└── next.config.js

Step 5: Home Page Component

Created the main page component, starting with a simple structure:

app/page.js 
                            
import Navigation from './components/Navigation'
import Hero from './components/Hero'
import About from './components/About'
import Skills from './components/Skills'
import Projects from './components/Projects'
import Contact from './components/Contact'

export default function Home() {
  return (
    <main>
      <Navigation />
      <Hero />
      <About />
      <Skills />
      <Projects />
      <Contact />
    </main>
  )
}

What I've Learned So Far

Even though I'm just getting started, I've already learned several important concepts:

  • JSX Syntax: Writing HTML-like syntax within JavaScript
  • Props: Passing data between components
  • File-based Routing: How NextJS automatically creates routes based on file structure
  • Image Optimization: Using NextJS's built-in Image component for better performance
  • CSS Modules: Scoped styling to prevent CSS conflicts
  • API Routes: Creating serverless functions within the NextJS app

Development Workflow

My current development process involves:

  • Planning components and their relationships
  • Creating reusable UI components
  • Testing components in isolation
  • Integrating components into pages
  • Optimizing for performance and SEO
  • Deploying to Vercel for testing

This NextJS migration represents a significant leap in my development journey. While there's still much to learn, the foundation is solid, and I'm excited to see how this more modern approach will improve both the development experience and the final product.
