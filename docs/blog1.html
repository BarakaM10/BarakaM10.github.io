The Setup Process

Setting up GitHub Pages was surprisingly straightforward after following the tutorial. Here's the step-by-step approach I took:

Step 1: Repository Creation

First, I created a new repository on GitHub. The naming convention is important - for a user site, it should be username.github.io. Replace my username with yours.

Terminal 
                            
# Clone the repository
git clone https://github.com/BarakaM10/BarakaM10.github.io.git
cd BarakaM10.github.io

# Create initial files
touch index.html
touch README.md

Step 2: Basic HTML Structure

I started with a simple HTML file to test the deployment:

index.html 
                            
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Maurice's Coding Journey</title>
</head>
<body>
    <h1>Welcome to My Coding Blog</h1>
    <p>This is where I'll document my learning journey!</p>
</body>
</html>

Step 3: GitHub Pages Configuration

In the repository settings, I navigated to the Pages section and configured:

  • Source: Deploy from a branch
  • Branch: main
  • Folder: / (root)

Step 4: Deployment Commands

Here are the git commands I used to deploy my first version:

Git Commands 
                                
# Add all files
git add .

# Commit changes
git commit -m "Initial blog setup with GitHub Pages"

# Push to main branch
git push origin main

Lessons

GitHub Actions

GitHub Pages automatically uses GitHub Actions for deployment. I can see the workflow in the Actions tab.

Static Site Hosting

GitHub Pages only hosts static sites - no server-side processing, so only HTML, CSS, and JavaScript.
